## 内容简介

### 简版

过去几年，人们谈论 AI 与软件工程时，最常见的问题是：模型会不会替代程序员。这本书认为，这个问题太早落在了“人会不会被替代”上，而忽略了更深的变化：**当模型开始稳定参与真实工作后，软件工程的重心正在从“亲手写实现”转向“设计可执行、可验证、可治理的工作系统”。**

本书围绕一个新兴但正在迅速成形的核心概念展开：`harness engineering`。如果说 prompt engineering 解决的是“如何把意图说清楚”，那么 harness engineering 解决的就是“如何把目标、上下文、工具、边界、验证、记忆与改进组织成一个让 agent 真能工作的系统”。这不是一本 AI 工具书，也不是一本 prompt 技巧书，而是一本关于 **Agent 时代软件工程、团队协作与组织能力如何重写** 的方法论书。

### 标准版

《从 Prompt 到 Harness：AI Agent 的工程化方法论》试图回答一个越来越绕不过去的问题：当模型已经能读仓库、调工具、改文件、跑验证并持续推进任务时，软件工程本身会被重写成什么样。

本书的核心判断是，AI 对软件工程的真正冲击，不只在模型能力，而在工作环境如何被重新组织。未来团队的差距，不会先体现在谁更早接入模型，而会先体现在谁更早把工作现场整理成 agent 也能稳定进入的系统。也正因为如此，工程师的价值不会简单消失，而会越来越多地上移到另一层：定义目标、组织知识、接入工具、建立验证、设计边界、安排交接，并把失败写回环境。

全书分为七篇正文和一篇案例篇。前半部分建立语言与结构：解释为什么 prompt engineering 已经不够，为什么 harness engineering 是被工程现场逼出来的新语言，并将 harness 拆解为七层结构。中段进入软件工程现场、验证与控制系统、组织与管理现实，讨论 repo 如何演化为 agent 的操作系统，测试如何扩展成执行系统的验证，组织如何用试点矩阵、90 天路线图、RACI、ROI 与 build vs buy 判断真正落地。后半部分则讨论这套经验向 coding 之外外溢的条件与边界，以及它究竟改变了什么、没有改变什么。案例篇汇集 OpenAI、Anthropic、LangChain、METR 等公开材料，并补入明确披露的教学性推演，让全书的方法论有可以反复回查的证据底盘。

这本书不是在追逐某个工具或某个热词，而是试图提出一种更长久的工程视角：**软件工程正在第一次稳定地服务于两类执行者，人和 agent；从这一刻起，决定系统上限的，将越来越不是单次实现本身，而是工作环境能否被写成可执行、可验证、可治理、可交接的结构。**

### 编辑参考卖点

- 它不是 AI 入门书，而是一本关于 **Agent 时代软件工程重构** 的方法论书。
- 它同时覆盖 **工程结构、验证控制、组织落地、失败边界与未来判断**，而不是停留在工具技巧层。
- 它既有 OpenAI、Anthropic、LangChain、METR 等公开现场，也明确区分事实、推演与判断，保持方法论写作的严谨性。
- 它试图把当下最前沿的实践，压成未来几年仍可被引用的长期判断。
